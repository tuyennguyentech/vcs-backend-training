# PostgreSQL Transaction

**Tóm tắt:** Trong bài này, bạn sẽ học cách xử lý các transaction trong PostgreSQL bằng các câu lệnh `BEGIN`, `COMMIT`, và `ROLLBACK`.

---

## Transaction trong cơ sở dữ liệu là gì?

Một transaction cơ sở dữ liệu là một đơn vị công việc duy nhất bao gồm một hoặc nhiều thao tác.

Ví dụ kinh điển của transaction là chuyển tiền ngân hàng từ tài khoản này sang tài khoản khác. Một transaction hoàn chỉnh phải đảm bảo sự cân bằng giữa tài khoản người gửi và người nhận — nghĩa là nếu người gửi chuyển X, người nhận phải nhận đúng X, không hơn không kém.

Một transaction trong PostgreSQL có tính **Atomic (Nguyên tử), Consistent (Nhất quán), Isolated (Độc lập), và Durable (Bền vững)** — thường được gọi tắt là **ACID:**

- **Atomicity (Nguyên tử):** Đảm bảo transaction được thực hiện theo kiểu "tất cả hoặc không có gì".
- **Consistency (Nhất quán):** Đảm bảo các thay đổi được ghi vào cơ sở dữ liệu là hợp lệ và tuân theo các quy tắc đã định.
- **Isolation (Độc lập):** Xác định mức độ ảnh hưởng của một transaction đến các transaction khác.
- **Durability (Bền vững):** Đảm bảo các transaction đã được commit sẽ được lưu trữ vĩnh viễn trong cơ sở dữ liệu.

---

## Thiết lập bảng mẫu

Tạo một bảng mới tên `accounts` để minh họa:

```sql
DROP TABLE IF EXISTS accounts;
CREATE TABLE accounts (
    id INT GENERATED BY DEFAULT AS IDENTITY,
    name VARCHAR(100) NOT NULL,
    balance DEC(15,2) NOT NULL CHECK(balance >= 0),
    PRIMARY KEY(id)
);
```

---

## Bắt đầu một Transaction

Khi bạn thực thi một câu lệnh, PostgreSQL tự động bọc nó trong một transaction ngầm định.

Để bắt đầu một transaction một cách tường minh, bạn dùng một trong các câu lệnh sau:

```sql
BEGIN TRANSACTION;
-- hoặc
BEGIN WORK;
-- hoặc đơn giản là
BEGIN;
```

Ví dụ, các câu lệnh dưới đây bắt đầu một transaction mới và chèn một tài khoản mới vào bảng `accounts`:

```sql
BEGIN;
INSERT INTO accounts(name, balance)
VALUES('Alice', 10000);
```

Trong phiên làm việc hiện tại, bạn có thể thấy sự thay đổi bằng cách truy vấn bảng `accounts`. Tuy nhiên, nếu bạn kết nối đến PostgreSQL ở một phiên khác và chạy truy vấn, bạn sẽ **không thấy** thay đổi đó — vì transaction chưa được commit.

---

## Commit một Transaction

Để áp dụng thay đổi vào cơ sở dữ liệu một cách vĩnh viễn, bạn commit transaction bằng:

```sql
COMMIT WORK;
-- hoặc
COMMIT TRANSACTION;
-- hoặc đơn giản là
COMMIT;
```

Sau khi thực thi `COMMIT`, PostgreSQL đảm bảo rằng thay đổi sẽ được bảo toàn ngay cả khi hệ thống gặp sự cố.

**Ví dụ đầy đủ:**

```sql
-- Bắt đầu transaction
BEGIN;

-- Chèn một hàng mới vào bảng accounts
INSERT INTO accounts(name, balance)
VALUES('Alice', 10000);

-- Commit thay đổi
COMMIT;
```

---

## Rollback một Transaction

Nếu bạn muốn hoàn tác các thay đổi, dùng câu lệnh `ROLLBACK`:

```sql
ROLLBACK;
-- hoặc
ROLLBACK TRANSACTION;
-- hoặc
ROLLBACK WORK;
```

**Ví dụ:**

```sql
-- Bắt đầu transaction
BEGIN;

UPDATE accounts
SET balance = balance - 1000
WHERE id = 1;

-- Hoàn tác thay đổi
ROLLBACK;
```

Sau khi rollback, dữ liệu trong bảng sẽ không thay đổi:

```
 id | name  | balance
----+-------+----------
  1 | Bob   | 10000.00
  2 | Alice | 10000.00
(2 rows)
```

Trong thực tế, bạn sẽ sử dụng transaction trong stored procedure của PostgreSQL cũng như trong mã ứng dụng như PHP, Java, hay Python.

---

## Tóm tắt

- Dùng `BEGIN` để bắt đầu một transaction tường minh.
- Dùng `COMMIT` để áp dụng các thay đổi vĩnh viễn vào cơ sở dữ liệu.
- Dùng `ROLLBACK` để hoàn tác các thay đổi đã thực hiện trong transaction.
